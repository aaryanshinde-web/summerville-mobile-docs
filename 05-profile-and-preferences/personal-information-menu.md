## Profile & Preferences: Personal Information Menu

> The single entry point for every contact-data action — phone, email, mailing address, residential address, and device management all live under one menu so members don't hunt for them.

### Step-by-Step Workflow

#### Step 1: Personal Information Menu

The menu lists five actions with their descriptive subtitles: **Phone Number — Update phone number**, **Email Address — Update email address**, **Mailing Address — Update mailing address**, **Residential Address — Update residential address**, and **Manage Devices — Update your linked devices**. Each row opens a dedicated screen for that action.

![Step 1: Personal Information Menu](../.gitbook/assets/Screenshot_1776963778.png)

### Summary

Keeping contact data fresh is the single most impactful member-self-service item — outdated phone numbers block OTP, outdated emails break alert delivery, and outdated addresses break card reissuance. Consolidating all five contact actions onto one menu means support can direct any contact-related question to a single navigation path ("Profile → Personal Information") instead of five different deep-links. Manage Devices lives here because it's the mobile analog of login security in the admin console — the member's phones and tablets that have been trusted for biometric login.

### Key Use Cases

* Member moves and needs to update mailing + residential: both rows are on the same menu, two taps apart.
* Branch staff updated phone on the core but OTP still failing: member opens Phone Number, pulls Refresh to sync.
* Member gets a new tablet and wants the old one deactivated: Manage Devices → revoke the old device from the list.
