## Profile & Preferences: Personal Information Menu

> The single entry point for every contact-data action — phone, email, mailing address, residential address, and device management all live under one menu.

### Step-by-Step Workflow

#### Step 1: Open the Side Menu

Tap the **☰** hamburger icon at the top-right of any screen.

![Step 1: Open the Side Menu](../.gitbook/assets/Screenshot_1776963761.png)

#### Step 2: Tap Settings

In the Side Menu, tap **Settings — Account and security settings**.

![Step 2: Tap Settings](../.gitbook/assets/Screenshot_1776963761.png)

#### Step 3: Tap Personal Information

Inside Settings, tap **Personal Information**. The Personal Information menu loads with five rows.

![Step 3: Tap Personal Information](../.gitbook/assets/Screenshot_1776963778.png)

#### Step 4: Pick Which Contact Action

The menu lists five actions with their descriptive subtitles:
- **Phone Number — Update phone number**
- **Email Address — Update email address**
- **Mailing Address — Update mailing address**
- **Residential Address — Update residential address**
- **Manage Devices — Update your linked devices**

Each row opens a dedicated screen for that action — see the individual docs for the full flow per action.

![Step 4: Pick Which Contact Action](../.gitbook/assets/Screenshot_1776963778.png)

### Summary

Keeping contact data fresh is the single most impactful member-self-service item — outdated phone numbers block OTP, outdated emails break alert delivery, and outdated addresses break card reissuance. Consolidating all five contact actions onto one menu means you can direct any contact-related question to a single navigation path (Side Menu → Settings → Personal Information) instead of five different deep-links. Manage Devices lives here because it's the mobile analog of login security in the admin console.

### Key Use Cases

* Member moves and needs to update mailing + residential: both rows are on the same menu, two taps apart.
* Branch staff updated phone on the core but OTP still failing: member opens Phone Number, pulls Refresh to sync.
* Member gets a new tablet and wants the old one deactivated: Manage Devices → revoke the old device from the list.
